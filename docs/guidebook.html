행동특성 및 종합의견과 세부능력 및 특기사항(세특)을 한 도구에서 작성합니다. 항목을 고르고 메모를 더하면 초안이 만들어지고, 글자수(바이트)를 실시간으로 확인하며, 반 전체를 엑셀로 한 번에 저장합니다.

2학생 등록추가/일괄

4생성개별/전체
5검토·저장엑셀
두 개의 탭화면 상단에서 행동특성 탭과 세부능력 및 특기사항(세특) 탭을 전환할 수 있습니다. 탭을 바꿔도 입력 내용은 보존됩니다. 아래 STEP 1~6은 행동특성 탭 기준이며, 세특 탭은 뒤쪽 ‘세특 탭 사용법’에서 따로 설명합니다.
0

최초 1회 · API 키 등록 (관리자 설정)

이 도구는 구글 Gemini로 문장을 만듭니다. 학교(또는 담당 선생님)가 키를 한 번만 넣어 두면, 이후 사용하는 모든 분은 키 입력 없이 바로 쓸 수 있습니다.

  1. 구글 AI Studio 접속 → aistudio.google.com/apikey 에서 API 키 만들기 (보통 AIza… 로 시작)
  2. 발급된 키를 복사
  3. 프로젝트 폴더의 .env.example 을 .env.local 로 복사한 뒤, 키 값을 붙여넣기
# .env.local — 이 파일은 git에 올라가지 않습니다(키 비공개 보관)
VITE_GEMINI_API_KEY=여기에_본인_키_붙여넣기

코드는 이 값을 자동으로 읽습니다. 모델 변경 등은 src/App.jsx 상단 AI_CONFIG 블록에서 합니다.

무료로 충분합니다Gemini Flash 계열은 무료 사용량이 넉넉해, 한 학교에서 한 학급~여러 학급을 처리하는 정도면 대개 비용이 들지 않습니다. 사용량이 매우 많아지면 유료 전환이 필요할 수 있습니다.
키를 코드에 못 넣는 상황이라면화면 우측 상단 키 입력을 누르고 키를 붙여넣으면 그 자리에서 바로 테스트할 수 있습니다. 단, 이 방식은 새로고침하면 사라지므로 정식 배포에는 위 코드 등록 방식을 쓰세요.
1

계열과 학년 선택 (행동특성 탭)

상단 설정 줄에서 계열(인문계 / 특성화고(공업계) / 특성화고(상업계) / 마이스터고 — 4가지 중 택1)과 학년(1~3)을 고릅니다. 선택에 맞춰 아래 체크 항목이 자동으로 바뀝니다.

  • 특성화고·마이스터고를 고르면 전공·실습, 진로·자격 항목이 추가됩니다.
  • 1학년은 적응, 2학년은 자치·주도, 3학년은 진로·진학(또는 취업) 항목이 붙습니다.
중간에 바꿔도 안전학생을 입력하다가 계열·학년을 바꾸면, 새 설정에 없는 체크는 자동으로 정리됩니다(공통 항목은 유지).
2

학생 등록

한 명씩 또는 반 전체를 한 번에 넣을 수 있습니다.

  • 학생 추가 — 빈 카드를 한 장씩 추가
  • 이름 일괄 추가 — 이름을 한 줄에 하나씩 붙여넣으면 번호가 자동으로 매겨지며 한꺼번에 생성됩니다(반 전체에 유용).
3

특성 체크 + 메모 입력

각 학생 카드에서 해당하는 특성 버튼을 눌러 선택하고, 아래 메모 칸에 구체적인 관찰 내용을 자유롭게 적습니다.

메모가 결과의 질을 좌우합니다체크만 하면 일반적인 문장이 나오고, 메모에 실제 사례(예: “학기 초 소극적이었으나 모둠 발표 이후 자신감 향상”, “또래 멘토링 자원”)를 적을수록 그 학생만의 구체적인 문장이 나옵니다.
4

생성하기

  • 카드의 생성 버튼 — 그 학생 한 명만 생성
  • 상단 전체 생성 — 입력된 모든 학생을 차례로 생성(결과가 끝나는 대로 표시)

마음에 안 들면 다시 생성으로 새 문장을 받을 수 있습니다. 결과는 그대로 두지 말고 다음 단계에서 검토·수정하세요.

5

글자수(바이트) 관리

생성된 문장 아래에 막대 게이지가 색으로 분량을 알려 줍니다.

여유 (제한의 80% 미만) 임박 (80% 이상) 초과 (제한 넘음)

한글 2B / 3B 토글과 제한(바이트)은 직접 조절합니다.

설정의미
3B (기본)현재 NEIS(UTF-8) 기준 — 한글 1자 = 3바이트
2B옛 기준 — 한글 1자 = 2바이트(일부 학교/항목)
제한항목별 허용 분량(기본 1500바이트). 학교 기준에 맞춰 숫자 변경
게이지가 기준입니다AI는 분량을 “대략” 맞추려 하지만 바이트를 정확히 세지는 못합니다. 최종 분량은 반드시 게이지를 보고 문장을 직접 다듬어 맞추세요.
6

검토하고 엑셀로 저장

  • 결과 칸을 클릭해 직접 수정할 수 있습니다(수정 즉시 글자수 재계산).
  • 카드 / 표 보기를 전환해 반 전체를 표로 훑어볼 수 있습니다.
  • 엑셀 저장을 누르면 .xlsx 파일이 내려받아집니다.

엑셀 컬럼: 번호 · 이름 · 계열 · 학년 · 행동특성 및 종합의견 · 글자수 · 제한초과(O).

세특 탭 사용법 (세부능력 및 특기사항)

상단 세부능력 및 특기사항 탭은 과목 단위로 작성합니다. 과목 설정은 한 번만 입력하면 그 과목 학생 전원(최대 25명)에게 공통 적용됩니다.

A

과목 만들기 · 유형 선택

  • 과목 추가로 여러 과목을 탭으로 관리합니다. 각 과목 탭의 ×로 삭제할 수 있습니다.
  • 과목 유형을 고릅니다 — 일반 교과(탐구·사고·개념 이해 중심) 또는 NCS 전문교과(직무 수행·기능 숙련 중심). 유형에 따라 활동 유형 후보와 문체가 달라집니다.
  • 과목명(예: 전기회로, 화학Ⅰ)과 제한(바이트)을 과목별로 설정합니다.
B

과목 공통 정보 입력 (한 번만)

  • 성취기준 / 능력단위·수행준거 — 일반 교과는 성취기준, NCS는 능력단위·수행준거를 입력합니다.
  • 수업 활동 유형 — 토론·발표·실험·프로젝트(일반) 또는 실습·직무 수행·안전관리(NCS) 등 버튼으로 선택, 기타는 직접 입력.
  • 공통 활동 내용 / 수행평가 기준 — 반 전체가 함께 한 활동·평가 맥락을 적습니다.
  • 교사 관점·서술 방향(선택) — 성장 중시, 진로 연계 등 서술의 초점을 지정합니다.
한 번 입력 → 25명 공통위 과목 정보는 학생마다 다시 쓸 필요 없이 그 과목 전원에게 자동 적용됩니다.
C

학생별 참여 수준 + 개별 메모 → 생성

  • 학생마다 참여 수준(주도적 / 적극적 / 보통 / 미흡)을 고르면 그 수준에 맞춰 톤이 차등 적용됩니다(‘미흡’도 부정적 낙인 없이 절제하여 서술).
  • 개별 메모에 그 학생만의 모습을 적을수록 결과가 구체화됩니다.
  • 행마다 생성 버튼, 또는 상단 이 과목 전체 생성으로 일괄 생성합니다. 이름 일괄 추가도 지원합니다(과목당 최대 25명).
D

엑셀로 저장 (전 과목)

엑셀 저장(전 과목)을 누르면 과목별로 시트가 분리된 하나의 .xlsx가 내려받아집니다.

엑셀 컬럼: 번호 · 이름 · 과목 · 참여수준 · 세부능력 및 특기사항 · 글자수 · 제한초과(O).

문제 해결

자주 나오는 상황과 해결법입니다.

생성을 눌러도 아무 일이 없거나 오류가 떠요
대개 API 키 문제입니다. ① 코드의 apiKey가 올바른지, ② 또는 우측 상단 키 입력에 키를 넣었는지 확인하세요. 키 옆에 ● 사용 가능 표시가 떠야 정상입니다.
“응답이 비어 있습니다”라고 나와요
모델명이 맞는지(gemini-2.5-flash 등), 또는 안전 필터에 걸릴 만한 표현이 메모에 없는지 확인하세요. 메모를 다듬고 다시 생성하면 해결되는 경우가 많습니다.
네트워크/연결 오류가 떠요 (브라우저에서 막힘)
브라우저에서 Gemini로 직접 요청하는 구조라, 일부 환경(보안 설정·사내망)에서 차단될 수 있습니다. 다른 브라우저나 네트워크로 시도하고, 정식 배포 시에는 안정적인 호스팅 환경에서 사용하세요.
잠시 잘 되다가 갑자기 막혀요 (429 / 사용량 초과)
무료 사용량의 분당 요청 한도를 넘은 것입니다. 전체 생성은 학생 사이에 간격을 두고 처리하지만, 인원이 많으면 잠시 기다렸다가 다시 시도하세요. 지속적으로 많이 쓴다면 유료 전환을 고려합니다.
결과가 글자수 제한을 자꾸 넘어요
정상입니다. 게이지가 빨강이면 문장을 줄여 직접 맞추거나, 제한값을 학교 기준에 맞게 올리세요. 또는 체크 항목을 줄여 다시 생성하면 짧아집니다.
